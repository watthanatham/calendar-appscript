# 📅 คู่มือใช้งาน: แปลงข้อความ Booking จาก LINE ลง Google Calendar อัตโนมัติ

> ฉบับคนทั่วไป — อ่านแล้วทำตามได้เลย ไม่ต้องมีพื้นฐานเขียนโปรแกรม

---

## 🎯 เครื่องมือนี้คืออะไร / ช่วยอะไร

ทุกวันนี้ที่ทำงาน รับ booking จาก LINE ต้องมานั่งกรอกลง Google Calendar เองทีละอัน เสียเวลาและพลาดง่าย

เครื่องมือนี้ช่วยให้:

1. **Copy ข้อความ booking จาก LINE** มาวางในไฟล์ Google Sheet
2. **กดปุ่ม 1 ครั้ง** → booking ถูกสร้างเป็น event ใน Google Calendar อัตโนมัติ
3. แยก**จำนวนตู้** ต่อ**สถานที่โหลด** (ปิ่นทอง/อมตะ/โหลด 2 ที่) ให้ชัดเจน
4. เก็บข้อมูล VGM, Invoice No., วันแจ้งรถรับตู้ ลงในรายละเอียดของ event

**ฟรี ไม่มีค่าใช้จ่าย ไม่ต้องลงโปรแกรมในเครื่อง**

---

## ⏱ ใช้เวลา setup ประมาณ 10–15 นาที (ทำครั้งเดียว)

หลังจากนั้น **ใช้งานประจำใช้เวลาแค่ 30 วินาที** ต่อครั้ง

- ✅ **ไฟล์ `Code.gs`** ที่ใช้กับระบบนี้ (ขอจากผู้ดูแลระบบ)

<a id="step1"></a>
## ✅ ขั้นที่ 1: สร้าง Google Sheet

1. เปิดเบราว์เซอร์ → ไปที่ **https://sheets.new** (จะสร้าง Sheet เปล่าให้ทันที)
2. ตั้งชื่อให้เข้าใจง่าย เช่น **"Booking Calendar"** (คลิกที่ "Untitled spreadsheet" มุมบนซ้ายเพื่อแก้ชื่อ)

> 💡 **เคล็ดลับ:** ใช้ Sheet เดียวซ้ำๆ ได้ตลอด ไม่ต้องสร้างใหม่ทุกครั้ง

---

<a id="step2"></a>
## ✅ ขั้นที่ 2: เปิดหน้า Apps Script

1. ในหน้า Sheet ที่เพิ่งสร้าง คลิกเมนู **Extensions** (อยู่บนแถบเมนูด้านบน)
2. คลิก **Apps Script**
3. จะมีหน้าใหม่เปิดขึ้นมา — นี่คือหน้าเขียนโปรแกรม

> 🤔 **หน้าตาเป็นแบบไหน?** จะเห็นไฟล์ชื่อ `Code.gs` ตรงกลาง มีโค้ดเริ่มต้น 2–3 บรรทัดที่ขึ้นต้นว่า `function myFunction()`

---

<a id="step3"></a>
## ✅ ขั้นที่ 3: ใส่โค้ดของระบบ

1. **ลบโค้ดเดิมทิ้งทั้งหมด** — คลิกในกล่องโค้ด กด `Ctrl + A` (เลือกหมด) แล้วกด `Delete`
2. **เปิดไฟล์ `Code.gs`** ที่ผู้ดูแลระบบส่งให้ (เปิดด้วย Notepad หรือโปรแกรมใดก็ได้ที่อ่านไฟล์ข้อความ)
3. กด `Ctrl + A` (เลือกข้อความทั้งหมด) แล้ว `Ctrl + C` (คัดลอก)
4. กลับมาที่หน้า Apps Script → คลิกในกล่องโค้ดที่ว่าง → กด `Ctrl + V` (วาง)
5. ตั้งชื่อ project: คลิกคำว่า **"Untitled project"** มุมบนซ้ายของหน้า Apps Script → ตั้งเป็น **"Booking Calendar"** → กด Rename

---

<a id="step4"></a>
## ✅ ขั้นที่ 4: เปิดบริการ Calendar API

ขั้นนี้สำคัญมาก ถ้าข้ามจะใช้งานไม่ได้

1. ในหน้า Apps Script มองที่**แถบซ้าย** จะเห็นคำว่า **Services** (มีเครื่องหมาย + อยู่ด้านขวา)
2. คลิกเครื่องหมาย **+** ข้างคำว่า Services
3. หน้าต่างจะเปิดขึ้นมา ให้ **เลื่อนหา "Google Calendar API"** (ใช้ Ctrl+F ค้นหาคำว่า Calendar ก็ได้)
4. คลิก **"Google Calendar API"** ให้เป็นไฮไลต์
5. ดูช่อง **"Identifier"** ด้านล่าง — ปล่อยให้เป็นค่าเริ่มต้น (จะเป็น `Calendar`)
6. กดปุ่ม **"Add"** มุมล่างขวา
7. กลับมาจะเห็นคำว่า **Calendar** เพิ่มขึ้นมาใต้ Services ในแถบซ้าย ✓

> ⚠️ **ถ้าไม่ทำขั้นนี้** จะเจอ error `Calendar is not defined` ตอนใช้งาน

---

<a id="step5"></a>
## ✅ ขั้นที่ 5: บันทึก และกลับไปที่ Sheet

1. กดปุ่ม **💾 Save** มุมบนของหน้า Apps Script (หรือ `Ctrl + S`)
2. รอจน text "Saving project..." หายไป (ประมาณ 2 วินาที)
3. กลับไปที่หน้า Google Sheet ที่เปิดค้างไว้
4. **กด F5 หรือ refresh หน้า Sheet** — สำคัญมาก! เมนูใหม่จะโผล่หลัง refresh
5. รอ Sheet โหลดเสร็จ — สังเกตที่แถบเมนูด้านบน จะมีคำว่า **"Booking"** เพิ่มขึ้นมาทางขวาสุดของเมนู (ถัดจาก Help)

> 🔄 **ถ้าไม่เห็นเมนู Booking** ให้ refresh อีกครั้ง หรือลองปิด-เปิด tab ใหม่

<a id="step6"></a>
## ✅ ขั้นที่ 6: ใช้งานครั้งแรก (อนุญาตการเข้าถึง)

ครั้งแรกที่ใช้ Google จะถามว่า "ยอมให้สคริปต์นี้เข้าถึง Calendar ของคุณไหม?" ต้องกดอนุญาตครั้งเดียว

1. ในหน้า Sheet ลอง**วางข้อความ booking สั้นๆ** ที่ช่อง **A1** เพื่อทดสอบ:
   ```
   BOOKING NO. 99999 = LOAD 1/1/2027 = จำนวน 1 ตู้โหลดอมตะ
   ```
2. คลิกเมนู **Booking** → **ส่งเข้า Google Calendar**
3. หน้าต่างขออนุญาตจะโผล่ขึ้นมา — ทำตามลำดับ:

   **(ก)** เลือกบัญชี Google ของคุณ

   **(ข)** ถ้าเจอหน้าจอที่เขียนว่า **"Google hasn't verified this app"** (สีเหลือง/เทา):
   - คลิก **"Advanced"** มุมล่างซ้าย
   - คลิก **"Go to Booking Calendar (unsafe)"** ที่โผล่มาด้านล่าง
   - ⚠️ **ปลอดภัย** เพราะเป็นสคริปต์ของคุณเอง (Google เตือนทุกสคริปต์ที่ไม่ได้ผ่านการรับรอง)

   **(ค)** หน้าจอใหม่จะแสดงสิทธิ์ที่จะขอ → คลิก **"Allow"**

4. รอ 2–3 วินาที จะมีกล่องเด้งขึ้นมาบอก:
   ```
   ✓ ส่ง 1 event เข้า Google Calendar
   BK 99999  01/01/2027  1 ตู้  @Amata
   ```
5. ไปเช็คใน Google Calendar (calendar.google.com) วันที่ 1 ม.ค. 2027 → จะเห็น event **"[Amata] Load 1 ตู้ — BK 99999"**

🎉 **ถ้าเห็น event แสดงว่า setup เสร็จสมบูรณ์!**

> 💡 ลบ event ทดสอบนี้ทิ้งได้เลย (คลิก event → กดถังขยะ)

## 📌 การใช้งานประจำ (หลังจาก setup เสร็จ)

หลังจากนี้ใช้งานง่ายมาก แค่ 3 ขั้น:

### 1️⃣ Copy ข้อความ booking จาก LINE

- กดค้างที่ข้อความ booking ใน LINE → เลือก **Copy**
- ถ้ามีหลาย booking copy ทีละอันแล้วเอามาต่อกันใน Sheet ได้

### 2️⃣ วางในช่อง A1 ของ Sheet

- เปิด Sheet ที่สร้างไว้ (`Booking Calendar`)
- คลิกที่ช่อง **A1**
- กด `Ctrl + V` วางข้อความ
- ถ้ามีหลาย booking → **คั่นแต่ละ booking ด้วยเส้น** `============` (ดูตัวอย่างด้านล่าง)

### 3️⃣ กดเมนู Booking → ส่งเข้า Google Calendar

- รอ 2–3 วินาที จะมีกล่องสรุปขึ้นว่าส่งเข้า calendar กี่ event
- ✅ เสร็จ! ไปเช็คใน Google Calendar ได้เลย

> 💡 **ลบข้อมูลเก่าใน Sheet ก่อนวางใหม่** เพื่อกันสับสน (เลือกทุกแถวที่มีข้อความ → กด Delete)
>
> 💡 **รันซ้ำ booking เดิมได้** ระบบจะ**อัปเดต event เดิม** ไม่สร้างใหม่ (กันลงซ้ำ)

<a id="format"></a>
## 📋 กฎการวางข้อความ Booking

ระบบนี้อ่านข้อความได้ฉลาด **copy จาก LINE มาวางทั้งดุ้นได้เลย** แต่มีกฎ 2 ข้อที่ต้องมี:

### ❶ ต้องมีคำว่า "BOOKING NO." + เลข booking

ตัวอย่างที่ใช้ได้:
- `BOOKING NO. 271382140`
- `BOOKING NO.45084065`
- `BOOKING NO.:W488092746` (มีตัวอักษรนำได้)

### ❷ ต้องมี "วันโหลด" — มีคำนำหน้าอย่างใดอย่างหนึ่ง:

- `LOAD 4/6/2026`
- `LOADING DATE = 10.06.2026`
- `โหลด 04-06-26`

วันที่ใช้ได้หลายแบบ:
- `4/6/2026` (วัน/เดือน/ปี ค.ศ.)
- `05.06.2026` (ใช้จุดคั่น)
- `04-06-26` (ปี 2 หลักได้ → 26 = 2026)

### 🔄 ถ้ามีหลาย booking — คั่นด้วยเส้น `====`

```
BOOKING NO. 11111 = LOAD 1/6/2026 = จำนวน 1 ตู้โหลดอมตะ
====================
BOOKING NO. 22222 = LOAD 2/6/2026 = จำนวน 2 ตู้โหลดปิ่นทอง
```

> 💡 เส้นจะมีกี่ตัว `=` ก็ได้ ขอแค่ ≥ 3 ตัว และอยู่บรรทัดของตัวเอง

### ✨ สิ่งที่ระบบดึงเพิ่ม (ใส่ก็ได้ ไม่ใส่ก็ได้)

| ข้อมูล | ใส่ในข้อความว่า | จะปรากฏใน Calendar |
|---|---|---|
| **จำนวนตู้** | `(3X20" FCL)` หรือ `1X20"` หรือ `จำนวน N ตู้` | Total + breakdown ต่อสถานที่ |
| **สถานที่โหลด** | `อมตะ` / `Amata` / `ปิ่นทอง` / `Pinthong` / `โหลด 2 ที่` | แยกเป็นบรรทัดในรายละเอียด |
| **เลข Invoice** | เช่น `1THA2605019, 2THA2605041` | บรรทัด Invoice No: ... |
| **VGM** | บรรทัดที่ขึ้นต้นด้วย `VGM` | บรรทัด VGM ในรายละเอียด |
| **คืนตู้หลังเที่ยงคืน** | คำว่า `คืนตู้หลังเที่ยงคืน` | ⚠ คืนตู้หลังเที่ยงคืน |

### 📝 ตัวอย่างที่วางได้เลย (ทดสอบทั้ง 3 booking พร้อมกัน)

```
BOOKING NO. 271381542 = LOAD 2/6/2026 = จำนวน 1 ตู้โหลดอมตะ และ จำนวน 1 ตู้โหลดปิ่นทอง
====================================================
BOOKING NO.49013513 (3X20" FCL)
LOADING DATE = 05.06.2026
VGM CUT OF : 05.06.2026 (ก่อน 15.00)
INVOICE NO. 2THA2605106 = 1X20" FCL โหลดที่สาขาปิ่นทอง
INVOICE NO. 1THA2605067 = 1X20" FCL โหลดที่สาขาอมตะ
INVOICE NO. 1THA2605068 = 1X20" FCL โหลด 2 ที่
====================================================
1X20 FCL - BOOKING NO.:W488092746 - โหลด 04-06-26 - โหลดที่ปิ่นทอง จำนวน 1 ตู้
1THA2605019, 2THA2605041
VGM.: คัต 16.00 : 04-06-26
```

วางแล้วกดเมนู → จะได้ event ใน Calendar 3 รายการ:
- **2 มิ.ย. 2026:** `[Amata+Pinthong] Load 2 ตู้`
- **5 มิ.ย. 2026:** `[Pinthong+Amata+โหลด 2 ที่(Both)] Load 3 ตู้`
- **4 มิ.ย. 2026:** `[Pinthong] Load 1 ตู้`

<a id="trouble"></a>
## 🔧 ปัญหาที่อาจเจอ + วิธีแก้

### ❌ ไม่เห็นเมนู "Booking" ใน Sheet

- **กด F5** refresh หน้า Sheet ใหม่
- ถ้ายังไม่เห็น → กลับไปที่ Apps Script ดูว่ามี error ไหม (จะมีสีแดง)
- ตรวจสอบว่า **บันทึก code แล้ว** (ปุ่ม 💾 Save)

### ❌ Error: `Calendar is not defined`

- ยังไม่ได้เปิดบริการ **Google Calendar API** → กลับไปทำ[ขั้นที่ 4](#step4)

### ❌ Error: `CalendarAPI is not defined`

- คุณตั้งชื่อ Identifier ของ service เป็น `CalendarAPI` แต่โค้ดต้องการ `Calendar`
- **วิธีแก้:** กลับไปที่ Apps Script → กดที่ Calendar service → ลบทิ้ง → Add ใหม่โดยใช้ Identifier `Calendar` (ค่าเริ่มต้น)
- หรือถ้าโค้ดเวอร์ชันใหม่ ระบบจะ**รับได้ทั้ง 2 ชื่อ** ไม่ต้องแก้

### ❌ กดเมนูแล้วเด้ง "Authorization required"

- ทำตาม[ขั้นที่ 6](#step6) → กดอนุญาตให้ครบทุกหน้า
- ถ้าเจอหน้าเตือน "unverified app" ให้กด Advanced → Go to ... (unsafe)

### ❌ Booking บางอันไม่เข้า Calendar

- ดูในกล่องสรุปหลังกดเมนู — จะมีหัวข้อ **"⚠ parse ไม่ได้"** บอกว่า booking ไหนพลาด
- เช็คว่า booking นั้นมี **"BOOKING NO." + เลข** และ **"LOAD/LOADING DATE/โหลด" + วันที่** ครบไหม

### ❌ วันที่ผิด (เช่นโผล่ปี 2026 แทน 2025)

- ระบบใช้รูปแบบ **วัน/เดือน/ปี** เสมอ — ถ้าเขียน `5/6/2026` หมายถึง **5 มิถุนายน 2026** (ไม่ใช่ 6 พฤษภาคม)
- เช็คว่าในข้อความเขียนปีถูกต้อง (ปีในรูปแบบ ค.ศ.)

### ❌ Event ลงผิดปฏิทิน (เช่น Calendar คนอื่นในบริษัท)

- โดย default ลงปฏิทินหลัก (Primary) ของบัญชี Google ที่ใช้
- ถ้าจะเปลี่ยน → **ติดต่อผู้ดูแลระบบเพื่อแก้ค่า `CALENDAR_ID`** ในโค้ด

### ❌ พิมพ์/วางข้อความใน Sheet แล้วบรรทัดเพี้ยน

- ลอง **paste ลง Notepad ก่อน** แล้ว copy จาก Notepad ไปวางใน Sheet
- หรือใช้ **Ctrl + Shift + V** (paste แบบ values only)
